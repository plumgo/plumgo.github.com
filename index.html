理解Python模块加载机制

待的解决问题：

  1. 项目中有多个层面的配置文件，希望能够通过Python的模块加载机制简化管理过程；

  2. 经常有需要重定义系统内建处理流程的需要，如异常处理，如果不明白模块加载机制，无法确定覆写方法的时机。

简介

Python至少提供了三种方法导入模块，import和from语句及__import__方法，常用的使用形式包括：

  1. import X：导入模块X并且创建模块名到当前的命名空间（可以通过as修改名称），导入后可以通过X.name引用模块中的方法。

  2. from X import *：导入模块X并且将模块中的所有public（除了”_”开始的所有东西）对象引入当前命名空间。所以，可以直接使用name调用X中的对象，但是X.name却不行，因为X未被定义。

  3. from X import a, b, c：导入模块X并且将指定的对象a，b，c载入命名空间，可以直接使用，其他状况与2相同。

  4. X = __import__(“X”)：载入方式与1相同，但是提供了通过模块名称载入的方式。

Sqlite的特别之处

SQLite提供完整的关系型数据库特性，由于常用于嵌入式场景，故有一些与传统SQL数据库不同的地方，官方网站上已经有概要介绍，这一篇补充几点操作层面上与众不同之处，主要内容翻译自apsw的文档，并不断更新。

游标

SQLite只会在请求之后计算每个结果行。例如查询返回1千万行记录，SQLite不会一次返回所有结果，而是通过游标逐条处理请求行。 在同一连接上的游标不是相互独立的。一个游标做出的任何变化，会体现在同一游标上的的所有其他游标上，事务中的游标也有这个特性。

绑定

绑定总是和游标一起使用。字符串拼接似乎更方便，但是会遇到很多麻烦，比如SQL注入攻击。

读书笔记：浪潮之巅

以前看过《IT通史》，也是一部气势磅礴的书。前半部非常赞，作者从结绳记事一直讲到差分机再到集成电路的发明，特别是对原理的通俗描述，读完真心感叹人类的智慧。如果说有什么不足，那就是到后半部讲IT公司发展史时，有点类似于国别体史书，主要着眼于决定公司命运的决策斗争，各种勾心斗角，足够细致，但缺一点宏观脉络，毕竟时间早，史料可能不够多。

《浪潮之巅》有点像纪传体史书，可以补上那后半部的遗憾。能将企业的兴衰规律联系起来，除了传统的观点，如公司文化，决策人素质，科技变革，政治形势等等《IT通史》中已经提到的因素，还有将这些因素串起来从而形成大势所趋的力量，力量之强，可以让微软的商业优势变成技术优势，也可以让Google的技术优势变成商业优势。这种力量就是生态链。
