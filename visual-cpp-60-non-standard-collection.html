Visual C++6.0未遵循标准部分

Visual C++ 6.0无疑是相当时间内C++编程的首选IDE。但编译器产生年代的关系和MS的战略问题，较之同类，对C++标准的支持程度不够好，很容易给编程带来困惑和不必要的麻烦，这一篇就用来收集VC6中一些与标准C++有出入的地方，并不断更新。

具体的解决问题办法可以去这里找：http://support.microsoft.com/?scid=ph;zh-cn;3003

<iostream>与<iostream.h>

较新的（相对于VC6产生的年代）C++标准反对把.h符号继续使用在标准头文件中，所以可以看到一般在较早的C/C++的标准中的标准库,如time.h在新标准中都有替代ctime。从功能角度来讲，仅仅支持字符流，以全局函数声明；而<iostream>中都是以STL形式声明的，通用性更强。这也可以同时解释，为什么<iostream>需要using namespace std;来引入命名空间。另外，输入输出流的C++标准规范接口在一些微妙的细节上都已经改进，所以推荐使用<iostream>。

<< >>不能声明友元函数

声明重载运算符<<或者>>为某个类友元，而后在函数中访问类中数据成员，会报错error C2248: 'var' : cannot access private member declared in class 'Clazz'，这个问题不是遵不遵从标准的问题，可以算是bug了。

解决办法有：

  1. 不使用友元函数，由类提供数据成员访问接口，如：GetVar()，这样可能会产生性能问题，需要将函数写成内联函数。

  2. 将友元函数的定义放到类里。

    不支持模板类分离实现 需要把.cpp中的声明和定义都放在.h中，另外类声明的时候只需要声明一次模板参数。

    更具体内容参看：http://support.microsoft.com/kb/239436

    变量在for循环中的作用域

在VC6中，上面语句中的i变量会在声明后一直存在，而新标准中会随for作用域结束后销毁。

默认类型为int

在VC6中如果不写类型，如const x = 0;会默认将x设置为int，而在新的标准中需要显示指定数据类型。
