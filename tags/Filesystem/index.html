Filebench：文件系统存储基准测试工具

介绍

Filebench是一个文件系统和存储基准测试工具。支持：

  1. 多种工作负载类型；
  2. 预定义40多种负载，可以模拟邮件、web、文件、数据库服务器行为；
  3. 通过Workload Model Language(WML)自定义工作负载；
  4. 多进程和多线程负载；
  5. 可设置的目录继承深度、宽度及文件大小，并符合统计分布；
  6. 异步I/O操作和进程同步原语；
  7. 集成吞吐量、延时、CPU使用系统调用的统计功能；
  8. 运行于Linux，FreeBSD，Solaris等POSIX兼容的操作系统。

这一篇介绍我自己写的一个工具FolderTracer。

首先，需要区别Trace（跟踪）和Monitor（监视）这两个动词，我认为Trace主要强调监视的长期和非入侵性，而Monitor侧重于观察细致和全面，Process Monitor就是这类，如果需要长期监控，恐怕此软件就会比较吃力（使用形式和开销都不能接受）。明确这个很重要，特别是在搜索英文资料，还有面对上面多种实现技术的选型的时候。

然后，需求是，一段时间内监控用户指定的文件夹（一般是工作目录）中文件的创建，修改，访问，删除操作，当用户修改文件是，可以大致知道修改量有多少。

Change Journal这名字挺直白。

很火的桌面搜索程序：Everything就是利用了NTFS系统的这个特性，通过读取和监控USN（后面会讲）而不是扫描文件来构建索引，所以搜索速度飞快，看来这个东西很好用。但是，相关的资料却是很少，特别是系统级介绍，而且一会USN，一会Change Journal，晕啊，找到代码都不敢用，还是老老实实做功课吧，找到了微软原始的两篇论文，来好好研究一下这个Change Journal。
